# Current Server Architecture

## Overview

`comment-bank-api/server.mjs` is now a thin application entrypoint. It loads
configuration, runs Umzug migrations, creates the Express app from
`src/app.js`, and starts listening on `config.port` (default `44344`).

The active backend is no longer a monolithic `server.mjs` file. Runtime code is
split across:

- `src/app.js`: Express app setup, CORS, session middleware, static files.
- `src/routes/index.js`: API route registration and request handlers.
- `src/models/index.js`: Sequelize model definitions and associations.
- `src/db/sequelize.js`: MariaDB/MySQL Sequelize connection.
- `src/db/migrate.js`: Umzug migration runner.
- `src/services/openai.js`: OpenAI client setup.
- `src/services/dbBackup.js`: guarded `mysqldump` backup/export helpers.

## Runtime Stack

- Node.js + Express 5.
- MariaDB/MySQL through Sequelize.
- Umzug migrations in `comment-bank-api/migrations/`.
- `connect-session-sequelize` for persistent sessions in the `Sessions` table.
- OpenAI Responses API through `openai.responses.parse`.
- Static frontend pages served from `comment-bank-api/public/`.

## Configuration

Environment variables are loaded from `comment-bank-api/.env`. The full list is
documented in `README.md` and mirrored in `comment-bank-api/.env.example`.

Important defaults:

- `PORT=44344`
- `DB_DIALECT=mariadb`
- `OPENAI_MODEL=gpt-5.2`
- `OPENAI_REASONING_EFFORT=none`
- `ENABLE_DB_BACKUP=false`

## Data Model

The core models are:

- `User`: username, bcrypt password hash, `isAdmin`.
- `Subject`: global subject list.
- `YearGroup`: global year-group list.
- `Category`: comment-bank category scoped to `userId`, `subjectId`,
  `yearGroupId`.
- `Comment`: comment text linked to a category.
- `Prompt`: per-user prompt text scoped to subject/year group.
- `SubjectContext`: per-user subject description and optional word limit scoped
  to subject/year group.
- `UserSubject`: user-selected visible subjects.
- `UserYearGroup`: user-selected visible year groups.

`Category`, `Prompt`, and `SubjectContext` are user-scoped. This matters for any
admin feature that writes comments for another staff member: the target staff
member's `User.id` must be used as the owner, not the admin's session user id.

## Main API Areas

### Auth and Sessions

- `POST /api/register`
- `POST /api/login`
- `POST /api/admin/login`
- `POST /api/logout`
- `GET /api/authenticated`
- `GET /api/user-info`
- `POST /api/change-password`

### Admin

- `GET /api/users`
- `POST /api/users`
- `DELETE /api/users/:username`
- `PUT /api/admin/user/:username/password`
- `GET /api/export-database`
- `POST /api/backup-database`
- Legacy duplicate admin routes also exist under `/api/admin/*` for subjects,
  year groups, users, export, and backup.

### Subjects and Year Groups

- `GET /api/subjects`
- `POST /api/subjects` (admin only)
- `PUT /api/subjects/:id` (admin only)
- `DELETE /api/subjects/:id` (admin only)
- `GET /api/year-groups`
- `POST /api/year-groups` (admin only)
- `PUT /api/year-groups/:id` (admin only)
- `DELETE /api/year-groups/:id` (admin only)

### Comment Banks

- `GET /api/categories-comments?subjectId=&yearGroupId=`
- `POST /api/categories`
- `PUT /api/categories/:id`
- `DELETE /api/categories/:id`
- `GET /api/comments?categoryId=`
- `POST /api/comments`
- `PUT /api/comments/:id`
- `DELETE /api/comments/:id`
- `POST /api/move-comment`

These routes currently operate on the logged-in user's comment bank.

### Prompts and Subject Context

- `GET /api/prompts?subjectId=&yearGroupId=`
- `GET /api/prompts/:subjectId/:yearGroupId`
- `POST /api/prompts`
- `PUT /api/prompts/:subjectId/:yearGroupId`
- `PUT /api/prompts/:id`
- `DELETE /api/prompts/:id`
- `DELETE /api/prompts/:subjectId/:yearGroupId`
- `GET /api/subject-context?subjectId=&yearGroupId=`
- `POST /api/subject-context`

Prompts and subject context are user-scoped.

### Report Generation and Imports

- `POST /generate-report`
- `POST /api/import-reports`
- `GET /api/export-categories-comments?subjectId=&yearGroupId=`
- `POST /api/import-categories-comments`

`/generate-report` and `/api/import-reports` are rate limited because they call
OpenAI. Imports from pasted reports merge with existing categories/comments for
the logged-in user. CSV import replaces the logged-in user's categories/comments
for the selected subject/year group.

## OpenAI and Privacy Controls

OpenAI calls use the Responses API with strict JSON schema output parsing.

Privacy safeguards currently in code:

- Pupil names are replaced with `PUPIL_NAME` before report imports and report
  generation prompts.
- `store: false` is sent with OpenAI requests.
- `safety_identifier` is a SHA-256 hash of the session user id.
- Subject descriptions are used as context and explicitly excluded from report
  output.

## Admin Uploads

The app already stores comment banks per staff user, subject, and year group,
and admin target-staff endpoints now allow an admin to import previous reports
for a selected staff user. These endpoints write categories/comments to the
target staff user's `userId`, not the admin user's `userId`, and successful
imports make the selected subject/year group visible to the target staff user.

See `docs/admin_staff_report_upload_plan.md` for implementation and deployment
safety details.
